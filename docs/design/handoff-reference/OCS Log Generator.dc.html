Help

What this tool does

It reads the Microsoft Forms export of staff training submissions and produces one AOIC-format log file per person. It tracks what’s been generated and sent in a ledger so nothing is done twice.

Sync procedure (§8.5)

  1. Download the latest ledger from the shared folder before you start.
  2. Drop it alongside the Forms export on the landing screen.
  3. Generate logs, then mark each as sent as you email them.
  4. Click Download updated ledger and replace the shared copy.

Manual path — OCS-Pivot (§2.6)

If a submission can’t be generated automatically, open the OCS-Pivot workbook, use the AOIC template tab, and copy the session rows in by hand. Then add the entry to the ledger manually.

For Claude Code — implementing this front end:

  • Tokens: use ocs-theme.css verbatim; do not hardcode hex. Dark mode = html[data-ocs-mode], default follows OS. Persist the toggle + text-size in localStorage.
  • Text size: stepper sets documentElement.style.fontSize (100/115/130%); all type uses rem so browser zoom stays safe.
  • A11y: WCAG 2.1 AA. Status uses symbol + text + color (never color alone). Real <button>/<input>, visible focus rings, drop zone is keyboard-operable.
  • Locked (do not redesign): AOIC xlsx layout (§6), filename convention, ledger schema, processing rules. This UI only orchestrates them.
  • Parsing: one submission = one person’s form (Name / Conference / Date(s) + a session table of Session / Date / Credit Hours / Relias flag). Sum hours; carry per-session warnings.
  • No network: all file handling client-side. No login, accounts, or settings pages — initials are the only identity input.
